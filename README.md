# Tool Mapper

Tool Mapper is a pilot internal web application developed as part of the **LF12a** module in my **Umschulung as Fachinformatiker für Anwendungsentwicklung**.

## Project Context

This project is being developed during the LF12a project phase from **16.03.2026 to 21.04.2026**.

The idea is based on a real organizational problem described to me by a friend working in the environment of **Infineon**. She explained that shared tools and objects in her team’s laboratory could not always be found easily, because there was no clear system to document their location, status, or current user.

ToolMapper is my attempt to design a practical solution to that problem.

## Project Goal

The goal of ToolMapper is to provide a simple internal system to:

- register shared laboratory tools
- track their current location
- store their default storage location
- manage lending and returns
- document tool status (e.g. available, borrowed, out of service)
- improve transparency and reduce time lost searching for missing tools

## Purpose

This project is intended as a **practical pilot application** for a specific use case in a laboratory context.  
At the same time, it serves as a technical and conceptual foundation that could later be adapted to similar tracking and inventory scenarios in other departments or organizations.

## Scope

The focus of this first version is on:

- internal use
- registered users
- shared lab tools
- traceability of movement and status
- clear and simple workflow

## Tech Stack

The exact implementation may evolve during the project, but the application is planned around technologies such as:

- **Python**
- **FastAPI**
- **HTML / CSS / JavaScript**
- **MySQL**
- **Git / GitHub** for version control and project documentation
